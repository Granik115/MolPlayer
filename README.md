# MolPlayer - Minecraft-style Audio Player

Простой, но красивый аудиоплеер для Windows 10/11 с интерфейсом в стиле молекулярного преобразователя из мода Advanced Solar Panels (Minecraft + IC2).

## Возможности (v1.0.3)
- Поддержка основных форматов: MP3, WAV, OGG, FLAC (через pygame)
- Создание и управление плейлистами
- Добавление треков из папки (выбор через проводник) — неаудио файлы игнорируются
- Воспроизведение с первого трека или в случайном порядке
- Перемотка (seek), пауза/воспроизведение с того же места
- Мини-плеер текущего трека (внизу справа)
- Сохранение плейлистов между запусками (в AppData)
- Регулировка громкости, отображение прогресса и длительности
- Автопереход к следующему треку

## Цветовая схема
Скопирована с интерфейса **Промышленный молекулярный преобразователь** (из мода Advanced Solar Panels / IC2):
- Глубокие синие и тёмно-синие тона (#0a1a2e, #215175 и т.д.)
- Яркий glowing cyan/blue (#00bfff, #3e80a3, #40e0d0) для рамок, прогресса и подсветки
- Лёгкая прозрачность всего окна (чтобы просвечивал рабочий стол)

## Запуск из исходников

```powershell
cd C:\projects\MolPlayer
python -m pip install -r requirements.txt

```

## Сборка в .exe (PyInstaller)
Используй готовый скрипт (рекомендуется onedir):

```powershell
cd C:\projects\MolPlayer
.\build.bat
# или
powershell -File build.ps1
```

Готовый exe + все зависимости: `dist/MolPlayer/MolPlayer.exe` (папка ~45-50 МБ).

### Переносимый архив для других устройств (рекомендуется)
После запуска `build.ps1` (или `build.bat`) в папке `releases/` автоматически создаётся готовый к пересылке архив:

```
releases/MolPlayer-v1.0.3-portable.zip
```

**Как раздать приложение:**
1. Запусти `powershell -File build.ps1` (или `.\build.bat`).
2. Возьми файл из `releases/MolPlayer-vX.Y-portable.zip`.
3. Перекинь этот zip на другую машину / другу / флешку.
4. Распакуй в любое удобное место.
5. Запусти `MolPlayer.exe`.

Ничего устанавливать не нужно — это полностью портативная версия.

### Антивирус ругается на MolPlayer.exe
Это классическая проблема всех приложений, собранных PyInstaller'ом.

**Почему так происходит:**
- PyInstaller собирает Python + библиотеки в "нестандартный" исполняемый файл.
- Антивирусы (особенно Windows Defender) очень агрессивно реагируют на такие файлы (false positive).

**Что делать пользователю при первом запуске:**
- Нажать **"Подробнее" → "Выполнить в любом случае"**.
- Или добавить папку с программой в исключения:
  - Параметры Windows → Конфиденциальность и безопасность → Безопасность Windows → Защита от вирусов и угроз → **Управление настройками** → **Исключения** → **Добавить или удалить исключения** → Добавить папку.

Мы уже используем лучшие практики:
- `--onedir` (папка, а не один огромный файл)
- `--noupx` (UPX сильно повышает вероятность блокировки)

Если человек совсем не хочет рисковать — можно дать ему исходный код + инструкцию ниже.

## Структура UI (как просил)
- **Вертикальное разделение**:
  - Слева: список плейлистов + кнопки создания/удаления
  - Справа: содержимое выбранного плейлиста
- **Горизонтальное в правой части**:
  - Верх: название плейлиста + действия (Воспроизвести, Случайно, Добавить папку, ...)
  - Большая нижняя область: список треков (с номером, названием, длительностью, кнопкой удаления)
- При воспроизведении: **маленькое подокно/панель внизу справа** (оверлей или docked в нижней части правой колонки) с текущим треком, прогресс-баром, кнопками управления и временем.

## Горячие клавиши (базовые)
- Space — Play/Pause
- Двойной клик по треку — воспроизвести
- Кнопки в мини-плеере
- Стрелки влево/вправо — перемотка на ±5 сек (в исходниках)

## Быстрый тест
В папке проекта есть `test_audio/` с тремя короткими тонами (сгенерированы при разработке).
1. Запусти `python main.py`
2. Создай плейлист кнопкой "+"
3. "📁 Добавить папку" → выбери `test_audio`
4. Нажми "▶ Воспроизвести" или "🎲 Случайно"
5. Проверь перемотку слайдером, паузу, переключение треков в мини-плеере (внизу справа над списком).

## Установщик и авто-обновления (v1.0+)

### Обычная установка (рекомендуется для большинства пользователей)
1. Скачай `MolPlayer-X.Y.Z-setup.exe` из Releases.
2. Запусти. Можно выбрать папку установки, создать ярлык на рабочем столе и т.д.
3. Приложение появляется в "Программы и компоненты" / "Установленные приложения".

### Портативная версия
Всё так же, как раньше — `MolPlayer-vX.Y.Z-portable.zip`. Распакуй и запускай `MolPlayer.exe` или `MolPlayer.bat` откуда угодно.

### Обновление прямо из приложения (без скачивания руками)
- Нажми кнопку **"Обновления"** в левой нижней части окна.
- Если есть новая версия на GitHub — приложение предложит скачать и установить её автоматически.
- После обновления программа перезапустится.
- Все настройки, плейлисты и состояние (последний трек, громкость, режим) сохраняются (хранятся в `%LOCALAPPDATA%\MolPlayer`).

Как готовить релиз на GitHub:
- Запусти `build.ps1` — получишь `releases/MolPlayer-vX.Y.Z-portable.zip` + `MolPlayer-portable.zip`
- (Опционально) Собери установщик:
  1. Установи Inno Setup (бесплатно).
  2. Открой `installer/MolPlayer.iss` в Inno Setup Compiler.
  3. Нажми "Compile" — установщик появится в `releases/`.
- На GitHub Releases создай новый релиз с тегом `vX.Y.Z`
- Прикрепи:
  - `MolPlayer-X.Y.Z-setup.exe` (если собирал)
  - `MolPlayer-vX.Y.Z-portable.zip`
  - `MolPlayer-portable.zip` (для авто-обновлений внутри приложения)

При обновлении через кнопку "Обновления" приложение ищет в релизе zip, содержащий слово "portable".

## Известные нюансы
- Точная перемотка на MP3 (особенно VBR) приблизительная (ограничение pygame). WAV/OGG точнее.
- При добавлении папки сканируются только файлы верхнего уровня (без рекурсии).
- Не-аудио файлы и повреждённые молча пропускаются.
- Прозрачность окна влияет на весь титулбар (особенность Tk).

## Примечания
- Для максимальной совместимости форматов рекомендуется иметь актуальный pygame.
- Если mp3 не играют — обновите pygame.
- Файлы, не являющиеся аудио, при добавлении папки пропускаются молча.
- Плейлисты хранятся в %LOCALAPPDATA%\MolPlayer\playlists.json
